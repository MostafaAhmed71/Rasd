## 0) ملاحظة مهمة قبل البدء: تحديث في هيكل الأدوار

النموذج المرئي يوضّح تسلسل أدوار **مختلف وأدق** مما ورد في `PRD.md` سابقًا. اعتمد الهيكل ده بدل القديم:

| الدور | المسمى في النموذج | الصلاحيات |
|---|---|---|
| المستوى الأول | **عضو هيئة تدريس** (Instructor) | يشوف مقرراته/شعبه فقط، يرصد الدرجات، يشوف جدوله ومراقبته، يقدّم طلبات اعتذار، عنده "مهام أخرى" (Checklist عام) |
| المستوى الثاني | **منسق/ة برنامج** (Program Coordinator) | يشوف كل أعضاء البرنامج المسؤول عنه فقط (مش قسم كامل — **برنامج** تحديدًا)، يشوف نسبة إنجاز كل عضو، **يقبل/يرفض طلبات الاعتذار** لأعضاء برنامجه (صلاحية فعلية، مش قراءة فقط) |
| المستوى الثالث | **المدير التنفيذي** (Executive Director) | نظرة عامة على كل البرامج والكلية، يعيّن منسقين للبرامج، يشوف إحصائيات شاملة (إجمالي الأعضاء، متوسط الإنجاز، طلبات الاعتذار الأسبوعية) |

**الفرق الجوهري عن التصميم السابق:** "منسق البرنامج" له صلاحية **تعديل فعلية** (قبول/رفض طلبات)، مش دور قراءة فقط زي ما كان مخطط لـ"رئيس القسم" قبل كده. اعتبر هذا هو المرجع النهائي للأدوار.

**بنية البيانات:** البرنامج (Program) يحتوي على أعضاء (Instructors)، وكل عضو يدرّس مقرر/شعبة أو أكثر. المقررات نوعين: **مقرر عادي** (درجات: أعمال سنة 40 + نصفي 20 + نهائي 40) و **تدريب ميداني** (درجات: مشرف ميداني 40 + مشرف أكاديمي 10 + 4 دورات منصات + تقرير + مناقشة). واجهة رصد الدرجات تتغيّر تلقائيًا حسب نوع المقرر المختار.

## 1) نظام التصميم (Design System)

### 1.1 الألوان
```
--color-primary-dark:   #1B4332   /* أخضر غامق — الشريط العلوي، الأزرار الأساسية، تفعيل القائمة الجانبية */
--color-primary:        #2D6A4F   /* أخضر متوسط — تدرّجات، حالات hover */
--color-primary-light:  #E8F3EE   /* أخضر فاتح جدًا — خلفية العنصر النشط في القائمة، خلفيات البطاقات الثانوية */

--color-bg-page:        #F7F8F7   /* خلفية الصفحات الداخلية (خارج البطاقات) */
--color-bg-card:        #FFFFFF   /* خلفية البطاقات والجداول */

--color-text-primary:   #1F2937
--color-text-secondary: #6B7280
--color-border:         #E5E7EB

--color-warning-bg:     #FDECC8   /* خلفية شارات "قيد المراجعة" / "بانتظار المراجعة" */
--color-warning-text:   #92400E

--color-success-bg:     #DCFCE7   /* خلفية شارة "نشط" */
--color-success-text:   #166534

--color-progress-low:   #D97706   /* لون شريط التقدّم لما النسبة منخفضة (مثال: أقل من ٦٠٪) */
--color-progress-high:  #2563EB   /* لون شريط التقدّم لما النسبة جيدة/مرتفعة */

--color-danger-text:    #DC2626   /* نص زرار "رفض" */
```

**ملاحظة:** الألوان أعلاه تقديرية بناءً على المعاينة البصرية للنموذج (مش قيم مستخرجة بدقة بكسل). قبل التسليم النهائي، لو متاح ملف تصميم أصلي (Figma) من مقدّم العرض، استخرج القيم الدقيقة منه واستبدل القيم هنا.

### 1.2 الخطوط
- خط عربي حديث يدعم الأرقام والاتجاه RTL بشكل كامل — رشّح: **IBM Plex Sans Arabic** أو **Tajawal** أو **Cairo** (الثلاثة متاحين مجانًا عبر Google Fonts ومناسبين لهوية حكومية/جامعية سعودية)
- العناوين: Bold
- النصوص الثانوية (Breadcrumb تحت العنوان، ملاحظات الجداول): حجم أصغر، لون رمادي `--color-text-secondary`

### 1.3 اتجاه الصفحة
- **RTL بالكامل** إجباري — القائمة الجانبية على اليمين، الجداول تبدأ من اليمين، الأزرار ومحاذاة النصوص كلها يمين

### 1.4 المكوّنات المتكررة (Components)

**البنية الأساسية لكل شاشة داخلية (بعد تسجيل الدخول):**
1. **شريط علوي أخضر غامق (Top Bar):** عنوان الصفحة يمين (Bold، أبيض) + سطر فرعي تحته (اسم المستخدم/الدور، أو مسار الصفحة) بخط أصغر وأفتح
2. **شريط ثانوي أبيض (App Header):** شعار المنصة واسمها يسار، وعلى اليمين: دائرة أفاتار بحروف اسم المستخدم (خلفية خضراء، نص أبيض) + الاسم + الدور + زرار "تسجيل الخروج"
3. **قائمة جانبية يمين (Sidebar):** عنوان "القائمة"، عناصر نصية بدون أيقونات، العنصر النشط له خلفية خضراء فاتحة + نقطة خضراء صغيرة على الحافة
4. **المحتوى الرئيسي:** عنوان الصفحة + سطر فرعي، ثم البطاقات/الجداول

**البطاقة (Card):** خلفية بيضاء، حواف دائرية، ظل خفيف (`box-shadow` بسيط)، padding مريح

**الجدول (Data Table):** رأس الجدول بخلفية خضراء غامقة ونص أبيض، صفوف بخلفية بيضاء، محاذاة نص يمين والأرقام وسط العمود

**زر أساسي (Primary Button):** خلفية `--color-primary-dark`، نص أبيض، حواف دائرية متوسطة — للأفعال الرئيسية (تسجيل الدخول، رصد الدرجات، قبول، إرسال الطلب)

**زر ثانوي (Secondary Button):** خلفية بيضاء أو رمادي فاتح، حدود رفيعة — للأفعال الثانوية (التفاصيل، إلغاء، تصدير Excel)

**زر رفض (Danger Button):** خلفية بيضاء، حدود ونص أحمر — تحديدًا لزر "رفض" في طلبات الاعتذار

**شريط تقدّم (Progress Bar):** رفيع، حواف دائرية، اللون يتغيّر حسب النسبة (منخفض = برتقالي/بني، مرتفع = أزرق) — **حدّد Threshold واضح في الكود** (مثلًا: أقل من 60% = برتقالي، 60% فأكثر = أزرق) وثبّته بدل تركه تخمينًا

**شارة حالة (Status Badge):** شكل حبّة دواء (pill)، خلفية لون خفيف حسب الحالة: "نشط" أخضر، "قيد المراجعة"/"بانتظار المراجعة" برتقالي/بيج

**دائرة الأفاتار (Avatar):** دائرة خضراء بحروف مختصرة من الاسم (مثال: "م.ع" لـ"مها عياد")

### 1.5 عناصر مستبعدة من التنفيذ الفعلي
الشاشات في الملف الأصلي فيها عنصرين خاصين بعرض النموذج فقط، **لا يُنفَّذان في التطبيق الحقيقي**:
- شارة ترقيم الشرائح أعلى اليسار ("1 / 12"، "2 / 12"...)
- تسمية سفلية "نموذج مرئي (Mockup) — بيانات تجريبية"

## 2) الشاشات بالتفصيل (12 شاشة)

### شاشة 1 — تسجيل الدخول
- خلفية كاملة بتدرّج أخضر غامق (Gradient) مع تأثير إضاءة دائرية خفيفة (Radial blur) في الخلفية
- بطاقة بيضاء في المنتصف: شعار الجامعة، اسم المنصة، سطر فرعي (اسم الكلية)
- حقل "اسم المستخدم" (Placeholder بصيغة username مثل khalaf.fadid)
- حقل "كلمة المرور"
- زر أساسي "تسجيل الدخول"
- فاصل "أو اختر حسابًا تجريبيًا للمعاينة"
- قائمة أزرار حسابات تجريبية: كل واحد فيه أفاتار + الاسم + الدور، بيسجّل الدخول مباشرة بضغطة واحدة (مفيد جدًا للعرض التقديمي وللتطوير، **يُنفَّذ فقط في بيئة Staging/Demo**، ويُستبعد أو يُخفى تلقائيًا في بيئة الإنتاج الحقيقية)

### شاشة 2 — المواد الدراسية (لوحة عضو هيئة التدريس الرئيسية)
- بطاقة لكل مقرر/شعبة يدرّسها العضو: عدد الطلاب (badge)، اسم المقرر، رمز المقرر — الشعبة، نسبة "الإنجاز" (Progress bar)، زرّي "التفاصيل" و"رصد الدرجات"
- القائمة الجانبية: المواد الدراسية (نشط) / رصد الدرجات / الجدول الدراسي / جدول المراقبة / تقديم طلب اعتذار / مهام أخرى

### شاشة 3 — رصد الدرجات (مقرر عادي)
- Dropdown لاختيار المقرر/الشعبة
- زر "تصدير Excel" أعلى يسار الجدول
- أعمدة: الرقم الجامعي، اسم الطالب، أعمال السنة (40)، نصفي (20)، نهائي (40)، المجموع (100)
- خلايا الدرجات القابلة للتعديل بحقول إدخال بيضاء داخل الجدول
- ملاحظة أسفل الجدول: المجموع يُحسب تلقائيًا فور الإدخال، مع تنبيه عند تجاوز الحد الأقصى لأي عمود

### شاشة 4 — رصد الدرجات (تدريب ميداني)
- نفس بنية شاشة 3، لكن الأعمدة: المشرف الميداني (40)، المشرف الأكاديمي (10)، دورة 1-4 (5 لكل واحدة)، كتابة التقرير (20)، مناقشة التقرير (10)، المجموع (100)
- بانر أخضر فاتح يظهر بعد التصدير: "تم تصدير الملف بنجاح"
- **ده تأكيد إضافي إن التوزيع اللي بنيناه سابقًا (ميداني+أكاديمي+منصات+تقرير+مناقشة) صحيح ومطابق للنموذج المعتمد**

### شاشة 5 — الجدول الدراسي
- عرض **بيانات منظّمة قابلة للبحث والفرز** (مش ملف PDF ثابت فقط) — جدول: اليوم، الوقت، الغرفة، رمز المقرر، اسم المقرر، الشعبة، الوحدات، العبء
- معلومات إضافية: اسم الملف الرسمي المصدر (لأغراض التوثيق)، تاريخ الرفع، الجهة (عمادة القبول والتسجيل)
- أزرار: "تنزيل الملف" (تحميل نسخة PDF/Excel) و"فتح في نافذة جديدة"

### شاشة 6 — جدول المراقبة
- نفس فكرة شاشة 5 لكن لبيانات مراقبة الاختبارات: اليوم، التاريخ، المقرر، الشعبة، القاعة، المراقب الأول، المراقب الثاني
- ملاحظة: نفس بيانات الجدول الرسمي الصادر من العمادة، لكن بشكل قابل للفرز والبحث بدل ملف PDF ثابت

### شاشة 7 — تقديم طلب اعتذار عن محاضرة
نموذج (Form) بالحقول:
- اسم المقرر (Dropdown)، رقم المقرر، الشعبة (تُملأ تلقائيًا حسب اختيار المقرر)
- التاريخ، الوقت
- **نمط المحاضرة:** حضوري / عن بُعد (Radio buttons)
- **نوع الاعتذار:** تقديم المحاضرة عن بُعد / الاعتذار نهائيًا (Radio buttons) — **دي نقطة مهمة: فيه فرق بين "بديل" (تحويل لأونلاين) و"اعتذار كامل"، لازم يُترجم لحالتين مختلفتين في قاعدة البيانات**
- أزرار: "إرسال الطلب" (أساسي) و"إلغاء" (ثانوي)

### شاشة 8 — مهام أخرى (Checklist عام)
- قائمة عناصر Checkbox (مهام إدارية عامة غير مرتبطة بمقرر محدد)، مثال: تحديث بيانات التواصل، تسليم تقرير منتصف الفصل، حضور اجتماع القسم، مراجعة توصيف مقرر
- العنصر المُنجز يظهر بخط مشطوب (Strikethrough) مع علامة صح خضراء
- **وحدة جديدة كليًا لم تُذكر في PRD السابق — يُضاف كجدول `general_tasks` مرتبط بـ `instructor_id`**

### شاشة 9 — أعضاء برنامجي (لوحة منسق/ة البرنامج)
- 3 بطاقات KPI أعلى الصفحة: عدد الأعضاء، متوسط إنجاز الرصد (%)، طلبات اعتذار معلّقة (عدد)
- بطاقة لكل عضو في البرنامج: أفاتار، الاسم، اسم المقرر/الشعبة، حالة (نشط)، نسبة إنجاز الرصد (Progress bar)
- **لو عند العضو طلب اعتذار معلّق، يظهر مباشرة داخل بطاقته** مع تفاصيل الطلب وزرّي "قبول"/"رفض" بدون الحاجة يفتح صفحة تانية
- ملاحظة أسفل الصفحة: المنسق يرى أعضاء برنامجه فقط، ولا يظهر له أي عضو من برنامج آخر — بحكم الصلاحيات المفروضة على مستوى قاعدة البيانات

### شاشة 10 — طلبات الاعتذار (صفحة منسق/ة البرنامج المخصصة)
- قائمة كاملة لكل طلبات الاعتذار "قيد المراجعة" لأعضاء البرنامج فقط
- كل طلب: اسم العضو، تفاصيل المقرر/الشعبة/التاريخ/نمط المحاضرة، شارة "قيد المراجعة"، زرّي "قبول"/"رفض"

### شاشة 11 — نظرة عامة على الكلية (لوحة المدير التنفيذي)
- 4 بطاقات KPI: إجمالي البرامج، إجمالي الأعضاء، متوسط الإنجاز الكلي (مع شارة مقارنة "أفضل من الفصل الماضي")، طلبات اعتذار هذا الأسبوع
- جدول/قائمة كل البرامج: اسم البرنامج، عدد الأعضاء، نسبة إنجاز الرصد (Progress bar ملوّن حسب النسبة)

### شاشة 12 — كل البرامج وتعيين المنسقين
- شبكة بطاقات لكل برنامج: اسم البرنامج، اسم المنسق الحالي (أو "لم يُعيّن" لو مفيش)
- زر "تعيين منسق" (لو مفيش منسق) أو "عرض التفاصيل" (لو معيّن بالفعل)
- **هذه الشاشة تحتاج Modal/صفحة فرعية لاختيار عضو هيئة تدريس من قائمة وترقيته لمنسق برنامج — لم تُفصَّل في النموذج، يُقترح تصميم بسيط: قائمة منسدلة بأسماء الأعضاء المؤهلين + زر تأكيد**

## 3) تحديثات مطلوبة على قاعدة البيانات (فوق ما ورد في PRD.md)

```sql
-- تحديث الأدوار
alter table profiles
  drop constraint if exists profiles_role_check,
  add constraint profiles_role_check check (role in ('executive_director', 'program_coordinator', 'instructor'));

create table programs (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  coordinator_id uuid references profiles(id),  -- NULL يعني "لم يُعيّن"
  created_at timestamptz default now()
);

-- ربط الشعب بالبرنامج بدل "department" النصي
alter table sections add column program_id uuid references programs(id);

-- نوع المقرر يحدد أي جدول درجات يُعرض
alter table sections add column course_type text check (course_type in ('regular', 'field_training'));

-- درجات المقرر العادي (منفصلة عن جدول grades الخاص بالتدريب الميداني)
create table regular_grades (
  student_id uuid references students(id) primary key,
  coursework_score numeric check (coursework_score between 0 and 40),   -- أعمال السنة
  midterm_score    numeric check (midterm_score between 0 and 20),      -- نصفي
  final_score      numeric check (final_score between 0 and 40),       -- نهائي
  total_score numeric generated always as (
    coalesce(coursework_score,0) + coalesce(midterm_score,0) + coalesce(final_score,0)
  ) stored,
  updated_by uuid references profiles(id),
  updated_at timestamptz default now()
);

-- توضيح نوع طلب الاعتذار
alter table absence_requests add column request_type text check (request_type in ('remote_delivery', 'final_absence'));
alter table absence_requests add column lecture_mode text check (lecture_mode in ('in_person', 'remote'));

-- مهام عامة (شاشة 8)
create table general_tasks (
  id uuid primary key default gen_random_uuid(),
  instructor_id uuid references profiles(id),
  title text not null,
  is_done boolean default false,
  created_at timestamptz default now()
);
```

## 4) ملاحظات التنفيذ
1. ابنِ نظام التصميم (الألوان، الخطوط، المكوّنات في قسم 1) كمتغيرات CSS / Tailwind قبل الشاشات، ليبقى الاستخدام موحّداً
2. اعتمد تخطيطاً مشتركاً واحداً (Top Bar + App Header + Sidebar) لكل الشاشات الداخلية
3. واجهة رصد الدرجات (شاشة 3 و4) صفحة واحدة بجدول يتغيّر حسب `sections.course_type`
4. حسابات الدخول التجريبي لبيئة التطوير فقط، ولا تظهر في الإنتاج
5. راجع `PRD.md` وحدّث مسمّيات الأدوار لتطابق القسم (0) أعلاه
