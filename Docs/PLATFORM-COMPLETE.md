## 1. ما هي المنصة؟

منصة ويب رسمية لأعضاء هيئة التدريس تغطي:

1. **رصد درجات** التدريب الميداني
2. **جدول المراقبة** (عرض وتنزيل PDF)
3. **جدول الدراسة** (عرض وتنزيل PDF)
4. **طلب اعتذار عن محاضرة** مع موافقة مسؤول النظام
5. **إدارة البيانات** من لوحة المسؤول (استيراد، ربط، تصدير، مستندات، طلبات)

تهدف لاستبدال ملف Excel المشترك، مع عزل بيانات كل عضو تدريس عبر صلاحيات قاعدة البيانات (RLS).

## 4. الأدوار والصلاحيات

### مسؤول النظام (Admin)
- استيراد بيانات الطلاب والشعب من Excel/CSV
- ربط عضو التدريس بكل شعبة
- متابعة نسبة إنجاز رصد الدرجات
- تصدير النتيجة النهائية لكل الشعب
- رفع جداول المراقبة والدراسة (PDF) لكل عضو
- مراجعة طلبات الاعتذار (قبول / رفض + ملاحظة)

### عضو هيئة التدريس (Instructor)
- رصد درجات طلاب شعبه فقط (حفظ تلقائي)
- تصدير درجات شعبته إلى Excel
- عرض وتنزيل جدول المراقبة وجدول الدراسة الخاصين به
- إنشاء طلب اعتذار عن محاضرة ومتابعة حالته

> التسجيل الذاتي من الموقع ينشئ حساب **عضو تدريس** فقط. حساب المسؤول يُنشأ من لوحة Supabase مع `"role": "admin"`.

## 6. لوحة عضو هيئة التدريس — بالتفصيل

### 6.1 رصد الدرجات
- اختيار **الرقم المرجعي** (الشعبة)
- إدخال الدرجات مع حفظ تلقائي عند مغادرة الحقل
- مؤشرات: جاري الحفظ / تم الحفظ ✓ / خطأ !
- تصدير Excel للشعبة الحالية
- على الموبايل: بطاقات لكل طالب بدل الجدول العريض

#### بنود الدرجة (المجموع 100)

| البند | الحد الأقصى |
|--------|-------------|
| المشرف الميداني | 40 |
| المشرف الأكاديمي | 10 |
| دورة منصات 1 | 5 |
| دورة منصات 2 | 5 |
| دورة منصات 3 | 5 |
| دورة منصات 4 | 5 |
| كتابة التقرير | 20 |
| مناقشة التقرير | 10 |

### 6.2 جدول المراقبة / جدول الدراسة
- يعرض ملف PDF الذي رفعه المسؤول لهذا العضو فقط
- أزرار: **تنزيل الملف** · **فتح في نافذة جديدة**
- إن لم يُرفع الملف بعد: رسالة توضيحية

### 6.3 طلب غياب (اعتذار عن محاضرة)

**بيانات الطلب**
- الكلية (افتراضي: الكلية التطبيقية — رفحاء)
- اسم عضو التدريس (يُملأ تلقائياً)

**لكل محاضرة (يمكن إضافة أكثر من صف)**
- اسم المقرر *
- الشعبة
- رقمها (رمز المقرر)
- التاريخ *
- الوقت *
- نمط المحاضرة: حضوري / عن بُعد
- نوع الاعتذار: تقديم المحاضرة عن بُعد / الاعتذار نهائياً

**حالات الطلب:** قيد المراجعة · مقبول · مرفوض  
يظهر للعضو ملاحظة المسؤول إن وُجدت.

## 7. لوحة مسؤول النظام — بالتفصيل

### 7.1 تبويب «البيانات والدرجات»
- استيراد ملف Excel أو CSV
- تصدير النتيجة النهائية (شيت منفصل لكل شعبة)
- جدول الشعب مع:
  - الرقم المرجعي
  - رمز المقرر
  - اسم المقرر
  - اسم عضو التدريس
  - نسبة الإنجاز (عدد من رُصدت درجاتهم / الإجمالي)
  - قائمة منسدلة لربط المدرّس

#### أعمدة ملف الاستيراد (بالترتيب)
1. اسم الطالب  
2. الرقم الجامعي  
3. التخصص  
4. المقرر  
5. النوع  
6. الفترة  
7. اسم الدكتور  
8. الرقم المرجعي  
9. رمز المقرر  

### 7.2 تبويب «المستندات»
- اختيار عضو التدريس
- اختيار النوع: جدول المراقبة أو جدول الدراسة
- رفع ملف PDF
- قائمة المستندات المرفوعة مع إمكانية التنزيل

### 7.3 تبويب «طلبات الاعتذار»
- فلترة حسب الحالة
- عرض تفاصيل المحاضرات
- قبول أو رفض مع ملاحظة اختيارية

## 8. قاعدة البيانات (Supabase)

### الجداول

| الجدول | الوظيفة |
|--------|---------|
| `profiles` | المستخدمون وأدوارهم |
| `sections` | الشعب / الأرقام المرجعية |
| `students` | الطلاب |
| `grades` | درجات كل طالب |
| `instructor_documents` | مستندات المراقبة والدراسة |
| `absence_requests` | طلبات الاعتذار |
| `absence_request_lectures` | صفوف المحاضرات داخل كل طلب |

### ملفات الترحيل (Migrations)
1. `001_initial_schema.sql` — الجداول الأساسية + RLS  
2. `002_add_course_code.sql` — رمز المقرر  
3. `003_add_instructor_name.sql` — اسم الدكتور من الاستيراد  
4. `004_documents_and_absence.sql` — المستندات + الاعتذار + Storage  
5. `005_drop_absence_head_semester.sql` — حذف رئيس القسم والفصل الدراسي  
6. `006_fix_policies_idempotent.sql` — إصلاح سياسات RLS  

### التخزين (Storage)
- **Bucket:** `instructor-documents` (خاص)
- **مسار الملف:** `{معرّف_العضو}/{نوع_المستند}/{وقت}.pdf`

## 9. الاستيراد والتصدير

### الاستيراد
- الصيغ: `.xlsx` · `.xls` · `.csv`
- يحدّث أو ينشئ الشعب والطلاب
- يحاول مطابقة اسم الدكتور مع حسابات الأعضاء المسجّلة

### التصدير
- مسؤول: ملف واحد لكل الشعب  
  `درجات-التدريب-الميداني-YYYY-MM-DD.xlsx`
- عضو تدريس: شعبة واحدة  
  `درجات-مرجع-{الرقم}-YYYY-MM-DD.xlsx`
- التصميم يحاكي شيت الجامعة (ترويسة، دمج خلايا، ألوان الأعمدة، اتجاه RTL)

## 10. المصادقة والأمان

- دخول بالبريد وكلمة المرور
- إنشاء حساب عضو تدريس من صفحة التسجيل
- عزل البيانات عبر **Row Level Security**:
  - العضو يرى/يعدّل شعبته فقط
  - المسؤول يرى ويدير الكل
- المستندات محمية: كل عضو يرى ملفاته فقط

## 11. تجربة الاستخدام (PWA والموبايل)

- يعمل كتطبيق من المتصفح (كمبيوتر / آيباد / جوال)
- يمكن تثبيته على الشاشة الرئيسية
- واجهة عربية واتجاه RTL
- جداول على الشاشات الكبيرة وبطاقات على الموبايل
- تحديث تلقائي للبيانات عند العودة للنافذة بعد التعديل

## 12. التشغيل والنشر

### متغيرات البيئة المطلوبة
```
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

### محلياً
```bash
cd app
npm install
npm run dev
```

### النشر
- **Vercel:** يبني من مجلد `app`
- **Netlify:** `base = app` و `publish = dist`
- يجب ضبط نفس متغيرات البيئة في لوحة الاستضافة

### بعد إنشاء مشروع Supabase
شغّل ملفات SQL بالترتيب من مجلد:
